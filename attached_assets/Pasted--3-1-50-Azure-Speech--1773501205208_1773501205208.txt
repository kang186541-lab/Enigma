데일리 코스에 3가지 문제가 남아있어.

## 문제 1: 녹음은 되지만 음성 인식 정확도가 낮음 (50% 정도)

녹음 시간은 충분한데 Azure Speech가 음성을 제대로 인식하지 못함. 발음 연습 화면에서는 거의 100% 인식되니까, STEP 3/4에서 Azure로 보내는 방식이 다른 것 같아.

### 반드시 확인할 것들

**1. Azure Speech 언어 설정 확인 (가장 유력한 원인)**

Azure Speech에 음성을 보낼 때 인식 언어(locale)를 설정해야 해. 이게 유저의 targetLang과 맞아야 함.

발음 연습 화면에서 어떤 locale을 쓰는지 확인하고 STEP 3/4에서도 동일하게 설정해줘:

```javascript
// targetLang에 따라 Azure 인식 언어를 설정
const localeMap = {
  en: 'en-US',   // 또는 en-GB
  es: 'es-ES',
  ko: 'ko-KR',
};

// Azure Speech 설정 시
speechConfig.speechRecognitionLanguage = localeMap[targetLang];
```

만약 STEP 3/4에서 이 설정이 빠져있거나 잘못된 언어로 설정되어 있으면 인식률이 매우 낮아져.

**2. 오디오 포맷 확인**

발음 연습에서 쓰는 녹음 옵션과 STEP 3/4의 녹음 옵션을 비교해서 동일하게 맞춰줘:

```javascript
// 발음 연습 화면의 Recording 옵션을 찾아서 여기에 그대로 복사
// 특히 다음 값들이 동일한지 확인:
// - sampleRate (16000 권장)
// - numberOfChannels (1 = mono 권장)  
// - bitRate
// - outputFormat (wav/webm 등)
```

Azure Speech는 16kHz, mono, 16bit PCM을 선호해. 다른 포맷이면 인식률이 떨어져.

**3. TTS 재생 후 오디오 모드 전환**

루디 TTS 재생 후 녹음 모드로 제대로 전환하고 있는지 확인:

```javascript
// 루디 TTS 재생 완료 후
sound.setOnPlaybackStatusUpdate((status) => {
  if (status.didJustFinish) {
    setTimeout(async () => {
      // 오디오 모드를 녹음 모드로 전환 (이게 핵심!)
      await Audio.setAudioModeAsync({
        allowsRecordingIOS: true,
        playsInSilentModeIOS: true,
      });
      setMicEnabled(true);
    }, 800);
  }
});
```

발음 연습에서는 TTS가 없으니 이 전환이 필요 없지만, STEP 3에서는 TTS→녹음 전환이 매번 일어나니까 이게 빠지면 인식이 불안정해져.

**4. 공통 모듈로 통일**

가장 확실한 방법: 발음 연습에서 Azure Speech API를 호출하는 함수를 찾아서, STEP 3/4에서도 그 함수를 그대로 import해서 사용해줘. 새로 만든 코드가 있으면 삭제하고 기존 것으로 교체.

## 문제 2: STEP 전환 시 대화 내용이 사라지고 버튼만 보임

STEP 3(루디와의 대화)이 완료되면 대화 내용이 전부 사라지고 “다음 단계로” 버튼만 있는 빈 화면이 나옴. 대화 내용은 유지한 채로 아래에 버튼이 나타나야 해.

### 수정 방법

대화를 지우지 말고, 대화 끝에 완료 메시지 + 버튼을 추가해줘:

```
┌─────────────────────────────┐
│  (기존 대화 내용 그대로 유지)   │
│                              │
│  🦊 루디: Great talking!      │
│  See you next time!          │
│                              │
│  ┌───────────────────────┐   │
│  │  ✅ 미션 토크 완료!      │   │
│  │                       │   │
│  │  🗣️ 말한 문장: 8문장    │   │
│  │                       │   │
│  │  [다음 단계로 →]        │   │
│  └───────────────────────┘   │
│                              │
└─────────────────────────────┘
```

구현:

- STEP 3 대화가 끝나면 (GPT가 shouldEnd: true 보내면):
1. 대화 내용은 그대로 유지 (채팅 메시지 리스트 삭제하지 마)
1. 채팅 리스트 맨 아래에 “미션 토크 완료!” 카드를 추가
1. 그 카드 안에 말한 문장 수 + “다음 단계로 →” 버튼
1. 하단 입력창(마이크 버튼, 키보드)은 숨김
1. 자동 스크롤로 완료 카드가 보이게

```jsx
// 대화 종료 시
if (shouldEnd) {
  // 입력창 숨기기
  setShowInput(false);
  
  // 완료 메시지를 채팅 리스트에 추가
  addMessage({
    type: 'completion',  // 특수 타입
    text: '미션 토크 완료!',
    sentenceCount: totalSentences,
  });
  
  // 자동 스크롤
  scrollToBottom();
}

// 렌더링
{message.type === 'completion' && (
  <View style={{ padding: 20, backgroundColor: 'rgba(201,162,39,0.15)', borderRadius: 16, margin: 16 }}>
    <Text style={{ fontSize: 20, color: '#c9a227', textAlign: 'center' }}>✅ 미션 토크 완료!</Text>
    <Text style={{ color: '#fff', textAlign: 'center', marginTop: 8 }}>🗣️ 말한 문장: {message.sentenceCount}문장</Text>
    <TouchableOpacity 
      onPress={goToNextStep}
      style={{ backgroundColor: '#c9a227', padding: 16, borderRadius: 12, marginTop: 16 }}
    >
      <Text style={{ textAlign: 'center', fontWeight: 'bold' }}>다음 단계로 →</Text>
    </TouchableOpacity>
  </View>
)}
```

STEP 1, STEP 2 완료 시에도 같은 방식으로 해줘:

- 기존 화면 내용은 유지
- 아래에 완료 카드 + “다음 단계로” 버튼 추가
- 화면 전체를 교체하지 마

## 문제 3: STEP 3 대화 후 문장 읽기 채점이 안 됨

STEP 3에서 루디와의 자유 대화가 끝난 후, 문장을 읽고 녹음하는 파트가 있는데 여기서 채점이 제대로 안 돼.

### 확인하고 수정해줘

1. **이 파트에서도 Azure 발음 평가 API를 쓰고 있는지 확인**: 일반 음성 인식(speech-to-text)이 아니라 발음 평가(pronunciation assessment)를 사용해야 점수가 나와. 발음 연습 화면에서 쓰는 발음 평가 코드를 그대로 가져다 써.
1. **reference text 전달 확인**: 발음 평가에는 유저가 읽어야 할 원본 텍스트를 Azure에 같이 보내야 해. 이게 빠져있으면 채점이 안 돼.

```javascript
// 발음 평가 시 반드시 원본 텍스트를 함께 보내야 함
const assessmentConfig = {
  referenceText: "Hello, nice to meet you.",  // 화면에 표시된 문장과 동일해야 함
  gradingSystem: "HundredMark",
  granularity: "Phoneme",
};
```

1. **녹음 인식 문제와 동일한 원인일 수 있음**: 문제 1에서 말한 locale 설정, 오디오 모드 전환 문제가 여기서도 적용됨. 발음 연습에서 쓰는 발음 평가 함수를 이 파트에서도 동일하게 import해서 사용해줘.
1. **점수 표시**: 채점 성공 시 발음 연습 화면과 동일하게 점수(%) + 별(⭐) + 루디 피드백을 표시해줘.
