긴급 버그! 데일리 코스 STEP 3(미션 토크)에서 “말하기” 버튼을 눌러도 음성 인식이 전혀 안 돼. “루디가 잘 못 들었어요. 다시 말해주세요”가 계속 반복됨.

## 현재 상황

- STEP 3 화면에서 “말하기 🎤” 버튼을 누르면 녹음은 시작되는 것 같지만 음성 인식 결과가 없음
- “루디가 잘 못 들었어요. 다시 말해주세요 🦊”가 3번 연속 나옴
- 반면에 앱의 다른 화면(발음 연습, 말하기 탭)에서는 녹음이 정상 작동함

## 원인과 해결

다른 화면에서는 녹음이 되는데 이 화면에서만 안 되니까, 이 화면의 녹음 구현이 기존과 다르게 되어있을 거야. 다음을 확인해줘:

### 1. 기존 작동하는 녹음 코드를 찾아서 그대로 복사해 와

발음 연습 화면(PronunciationScreen 또는 비슷한 이름)에서 사용하는 녹음 관련 코드를 찾아줘. 거기서 다음을 확인해:

- 어떤 라이브러리 쓰는지 (expo-av? @react-native-voice? Azure Speech SDK 직접?)
- 녹음 시작/종료 함수
- Azure Speech API 호출 방식
- 음성→텍스트 변환 로직

그 코드를 STEP 3 화면에 그대로 가져다 써. 새로 만들지 마.

### 2. 흔한 원인들 체크

**오디오 세션 설정 누락:**

```javascript
// iOS에서는 오디오 세션을 녹음 모드로 설정해야 함
await Audio.setAudioModeAsync({
  allowsRecordingIOS: true,
  playsInSilentModeIOS: true,
});
```

이 설정이 STEP 3 화면에 없으면 추가해줘.

**마이크 권한 재요청:**

```javascript
const { status } = await Audio.requestPermissionsAsync();
if (status !== 'granted') {
  // 권한 없음 처리
}
```

**녹음 객체가 제대로 생성 안 됨:**

```javascript
// 녹음 시작 전에 이전 녹음 객체를 정리해야 함
if (recording) {
  await recording.stopAndUnloadAsync();
}
const newRecording = new Audio.Recording();
await newRecording.prepareToRecordAsync(Audio.RecordingOptionsPresets.HIGH_QUALITY);
await newRecording.startAsync();
```

**Azure Speech API 키/엔드포인트가 이 화면에서 제대로 전달 안 됨:**
기존 발음 연습에서 사용하는 Azure 키와 엔드포인트(koreacentral)를 이 화면에서도 같은 값을 사용하는지 확인.

### 3. 가장 확실한 방법

기존에 작동하는 녹음+음성인식 로직을 공통 훅(hook)이나 유틸리티로 만들어서 재사용해줘:

```javascript
// hooks/useSpeechRecognition.js (또는 비슷한 것이 이미 있을 수 있음)
export function useSpeechRecognition() {
  const startListening = async () => {
    // 기존 발음 연습에서 쓰는 녹음 시작 코드 그대로
  };
  
  const stopListening = async () => {
    // 기존 발음 연습에서 쓰는 녹음 종료 + Azure 전송 코드 그대로
    return recognizedText;
  };

  return { startListening, stopListening, isListening, recognizedText };
}
```

이 훅을 STEP 3 화면에서 import해서 사용하면 됨.

### 4. 디버깅

콘솔에 녹음 관련 에러가 있는지 확인해줘:

- `console.log`로 녹음 시작/종료 시점 로깅
- Azure API 응답 로깅
- 에러가 있으면 그 에러 메시지 기반으로 수정

### 5. 에러 시 fallback

녹음이 실패하면 키보드 입력으로 자동 전환해줘:

```
"루디가 잘 못 들었어요. 키보드로 입력해볼까요? 🦊"
[🎤 다시 말하기]  [⌨️ 키보드로 입력]
```

3회 연속 실패하면 자동으로 키보드 입력 모드로 전환하고, “음성 인식에 문제가 있어요. 키보드로 입력해주세요”라고 안내.

## 중요

이건 데일리 코스의 핵심 기능이 완전히 먹통인 거라 최우선으로 고쳐줘. STEP 3은 말하기가 전부인데 녹음이 안 되면 아무것도 할 수 없어.
