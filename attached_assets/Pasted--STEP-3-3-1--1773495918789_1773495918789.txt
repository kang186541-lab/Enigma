데일리 코스 STEP 3(실전 말하기) 화면에서 3가지 문제가 있어. 전부 수정해줘.

## 문제 1: 루디가 이모티콘과 마크다운 기호를 소리내어 읽음

루디의 대사에 😊 이모티콘이나 **굵은 글씨** 마크다운이 포함되면 TTS가 그걸 그대로 읽어버려. 예를 들어 “Nice to meet you 😊”를 “Nice to meet you smiling face with smiling eyes”로 읽음.

### 수정 방법

TTS로 보내기 전에 텍스트를 정리(sanitize)하는 함수를 만들어줘:

```javascript
function sanitizeForTTS(text) {
  return text
    // 이모티콘 제거
    .replace(/[\u{1F600}-\u{1F64F}]/gu, '')  // 얼굴
    .replace(/[\u{1F300}-\u{1F5FF}]/gu, '')  // 기호
    .replace(/[\u{1F680}-\u{1F6FF}]/gu, '')  // 교통
    .replace(/[\u{1F900}-\u{1F9FF}]/gu, '')  // 추가
    .replace(/[\u{2600}-\u{26FF}]/gu, '')    // 기타
    .replace(/[\u{2700}-\u{27BF}]/gu, '')    // 딩뱃
    .replace(/[\u{FE00}-\u{FE0F}]/gu, '')    // 변형 선택자
    .replace(/[\u{200D}]/gu, '')             // ZWJ
    // 마크다운 기호 제거
    .replace(/\*\*/g, '')     // **굵은 글씨**
    .replace(/\*/g, '')       // *기울임*
    .replace(/__/g, '')       // __굵은 글씨__
    .replace(/_/g, ' ')       // _기울임_
    // 연속 공백 정리
    .replace(/\s+/g, ' ')
    .trim();
}
```

이 함수를 루디의 대사를 TTS로 재생하는 모든 곳에 적용해줘. 화면에 표시할 때는 이모티콘 그대로 보여주고, TTS로 읽을 때만 제거.

그리고 화면에서도 마크다운 **기호**가 그대로 보이고 있어 (예: `**"I'm from Korea."**`). 마크다운을 파싱해서 실제 굵은 글씨로 렌더링하거나, 아니면 강조 텍스트는 골드색(#c9a227)으로 표시하고 마크다운 기호는 제거해줘.

강조 처리 방법:

- GPT 응답에서 **텍스트** 패턴을 찾으면 → 해당 텍스트를 골드색(#c9a227) + 약간 굵게 표시
- 마크다운 기호(**) 자체는 화면에서도 제거

## 문제 2: 루디의 목소리가 여자 목소리임

루디는 남자 여우 탐정인데 현재 여성 TTS 음성이 나오고 있어. 남성 음성으로 변경해줘.

### 수정 방법

데일리 코스의 STEP 3에서 루디 대사의 TTS 음성을 다음으로 변경:

|targetLang|현재 (여성)           |변경 (남성)               |
|----------|------------------|----------------------|
|영어        |en-GB-SoniaNeural |**en-GB-RyanNeural**  |
|스페인어      |es-ES-ElviraNeural|**es-ES-AlvaroNeural**|
|한국어       |ko-KR-SunHiNeural |**ko-KR-InJoonNeural**|

이 변경을 데일리 코스뿐 아니라 루디가 말하는 모든 곳에 적용해줘:

- 데일리 코스 STEP 1 (듣고 따라하기) — 이건 학습 문장이니 남녀 섞어도 괜찮지만, 루디가 직접 말하는 브리핑은 남성
- 데일리 코스 STEP 3 (실전 말하기) — 루디 대사는 남성
- 스토리 모드에서 루디 대사 TTS — 남성
- 미션 브리핑 TTS — 남성

NPC별로 음성을 구분해줘:

- 루디(남성 캐릭터) → 항상 남성 음성 (RyanNeural / AlvaroNeural / InJoonNeural)
- 여성 NPC (엘리너, 이사벨, 수진, 미스 페니 등) → 여성 음성
- 남성 NPC (톰, 카를로스, 돈 미겔, 민호 등) → 남성 음성

## 문제 3: 실전 말하기 화면에서 음성 녹음 인식이 안 됨

STEP 3 화면에서 “말하기 🎤” 버튼을 눌러서 녹음하면 음성이 인식되지 않아.

### 수정 방법

다음을 확인하고 수정해줘:

1. **마이크 권한**: STEP 3 화면에서 마이크 권한이 제대로 요청되고 있는지 확인. 다른 화면(발음 연습)에서는 녹음이 되니까 권한 자체는 있을 텐데, 이 화면에서 권한을 다시 체크하고 있는지 확인
1. **Azure Speech 연결**: 녹음 후 Azure Speech API로 음성→텍스트 변환이 되고 있는지 확인. 발음 연습 화면에서 사용하는 것과 동일한 Azure Speech 설정을 사용해야 해
1. **녹음 시작/종료 로직**:
- “말하기” 버튼을 누르면 녹음 시작
- 다시 누르거나 일정 시간 침묵하면 녹음 종료
- 녹음된 오디오를 Azure Speech로 전송
- 변환된 텍스트를 GPT-4o에 전달
1. **기존 발음 연습의 녹음 로직을 재사용**: 발음 연습 화면에서 이미 작동하는 녹음 + Azure Speech 코드를 그대로 가져다 써. 새로 만들지 말고 같은 함수/모듈을 import해서 사용해줘.
1. **에러 핸들링**: 녹음 실패 시 “루디가 잘 못 들었어요. 다시 말해주세요 🦊” 메시지 표시하고 재시도 버튼 제공
1. **로딩 상태**: 녹음 후 Azure 분석 중일 때 “루디가 듣고 있어요… 🦊” 표시 (이전에 수정한 것처럼 “Azure 분석 중” 아닌 루디 표현으로)
