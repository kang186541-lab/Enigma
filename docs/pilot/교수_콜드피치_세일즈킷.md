# 교수 콜드피치 세일즈킷 — "수업 밖 말하기 과제 측정 도구"

> 목적: **이번 주에 교수 1명에게서 "6주 과제로, 성적 반영해서 쓰겠다"는 약속을 받아온다.**
> 만들 것은 앱 기능이 아니라 이 피치 + [주간 리포트 목업](교수_주간리포트_목업.html) 한 장.
> 근거: 5모델 교차검증 + 프로그램-레벨 해부 결론(2026-06-29). 진짜 위험은 음성모델이 아니라 *아무도 원치 않는 걸 만드는 것*.

## 0. 포지션 (이 한 줄만 외운다)

> **"학생들 수업 밖 말하기 과제를, 음성 하나하나 안 듣고도 누가 했는지 확인하고 성적 근거로 쓰게 해드립니다."**

- 파는 것 = **수행 증거 + 채점 시간 절감** (○)
- 파는 것 아님 = "AI가 자동 채점/발음 점수" (✕ — 신뢰성 책임만 떠안고 산다)
- 경쟁자 = 스픽/듀오링고 (✕) → **"지금 교수님이 하는 방식: 안 함 / 일일이 듣기 / 종강날 몰아 채점"** (○)

**한 문단 버전:**
"교양영어·회화 수업에서 '수업 밖에 말하기 연습해와라'는 과제는 내기는 쉬운데 *확인*이 지옥입니다. 학생 40명 음성을 하나씩 들을 수도 없고, 안 들으면 누가 진짜 했는지 모릅니다. 저희는 학생이 매일 5문장씩 말하면 **누가 며칠 했고, 얼마나 말했고, 수상한 학생이 누군지**를 주간 리포트 한 장 + CSV로 드립니다. 교수님은 애매한 학생 몇 명 녹음만 직접 들으시면 됩니다. 6주 무상 파일럿으로, 학생 부담·설치 없이(웹 링크) 한 분반만 돌려보시겠어요?"

## 1. 아웃리치 3종 (상황별로 골라서)

### A. 카톡/문자/DM (3줄 — 아는 교수/조교/지인 소개용)
```
교수님, 교양영어 '수업 밖 말하기 과제' 확인을 자동화하는 도구를 만들고 있습니다.
학생이 매일 5문장 말하면 누가 며칠 했는지·발화량을 주간 리포트로 드려요(음성 일일이 안 들으셔도 됩니다).
한 분반만 6주 무상으로 돌려보실 수 있을까요? 리포트 샘플 1장 보내드릴게요.
```

### C. 대면/줌 15분 스크립트
1. **(2분) 문제 공감** — "수업 밖 말하기 과제, 확인 어떻게 하세요? 음성 다 들으세요, 아니면 사실상 못 보세요?" → *그들이 페인을 말하게 둔다.*
2. **(3분) 리포트 목업 1장** 보여주기 — "이게 매주 가는 리포트입니다. 76% 제출, 미제출 5명, 그리고 이 학생은 제출은 했는데 발화량이 이상해서 '확인 필요'로 떴어요. 교수님은 이 몇 명만 들으시면 됩니다."
3. **(2분) 학생 경험** — 웹 링크 → 5문장. 설치·비용 없음.
4. **(3분) 정직한 한계** — "자동 채점 아닙니다. 점수 가중치는 교수님이 정하시고, 저흰 증거만 모읍니다. 그래서 억울한 학생 항의에도 '실제 녹음 기록'으로 방어됩니다."
5. **(5분) 클로징** → 아래 '받아야 할 답'을 *명시적으로* 받는다.

## 2. 받아야 할 답 (진짜 신호 ✅ vs 쓰레기 ❌)

✅ **진짜 신호 (하나라도 받으면 성공):**
- "다음 학기/이번 학기 **이 수업**에 과제로 넣겠다"
- "학생들에게 **설치(접속)하게 하겠다**"
- "성적/참여 점수에 **반영하겠다**"
- "주간 리포트를 **받아보겠다**"
- "잘 되면 **다음에도 쓰겠다**" / "소액이라도 **예산 잡아보겠다**"

❌ **쓰레기 신호 (받아도 0점 — 속지 말 것):**
- "좋네요", "관심 있어요", "나중에 연락주세요", "잘 되길 바랍니다", "자료 보내주세요"(만 하고 끝)

> **규칙:** 미팅 끝에 *"그럼 ○월 ○주차에 이 분반에 과제로 넣는 걸로, 제가 수업 코드 발급해 드릴게요"* 까지 못 가면 → 그 교수는 "예"가 아니다.

## 3. 자주 나올 반론 + 답 (Fugu 신뢰성 체크)

| 교수의 질문 | 답 (정직하게) |
|---|---|
| "대리 녹음/친구가 대신하면?" | 막진 못합니다. 단 학번·접속기기·발화 패턴이 기록돼 *수상한 케이스를 '확인 필요'로 띄웁니다.* 완벽 방지가 아니라 '지금(안 함)보다 낫다'가 목표입니다. |
| "그냥 읽기만 해도 말하기로 치나?" | 발화량·시간으로 '읽기만 한 듯한' 케이스를 표시합니다. 단 '제대로 말했는지'의 최종 판단은 교수님 몫 — 그래서 '확인 필요 녹음'만 들으시면 됩니다. |
| "이 점수 믿고 성적 줘도 되나?" | **점수를 드리는 게 아니라 '수행 기록'을 드립니다.** 성적 반영 여부·가중치는 교수님이 정하세요. 권장은 '참여/완료율' 반영(절대 점수 X). |
| "억울한 학생이 항의하면?" | 학생별 **실제 녹음·제출 일시·발화량 로그**가 남아 근거로 방어됩니다. (지금은 그 근거조차 없습니다.) |
| "개인정보·음성 녹음, 학교가 허용하나?" | 학생 **동의 기반**, 학번 마스킹, 음성 원본은 교수 확인용 임시 보관 후 **종료 시 삭제**. 동의서 양식 드립니다. 도입 전 학과에 확인해 주세요. |
| "수업 시간 잡아먹나?" | 오리엔테이션 5분 1회뿐. 이후 전부 수업 밖 자율. 교수님은 주 1회 리포트만. |
| "학생 40명 다 봐야 하나?" | 아니요. 리포트가 **미제출·확인필요 학생만 위로 띄웁니다.** 나머지는 안 보셔도 됩니다. |

## 4. 돈 얘기 (결제 구현은 미뤄도, 이 질문은 지금)
파일럿은 무상. 단 종료 인터뷰에서 *반드시* 묻는다:
- "다음 학기에도 쓰시겠어요? (예/아니오 + 이유)"
- "쓴다면 누가 비용을 낼까요 — 교수님 연구비? 학과/어학원 예산? 학생 개인?"
- "학생당 월 ___원이면 합리적일까요?"
> 무상 파일럿만 반복 = 또 다른 자기위안. 지불 주체·금액 감을 잡는 게 목표.

## 5. 타깃 교수 찾기 (첫 10명 리스트)
우선순위 높은 순:
1. **경상국립대 어학교육원 / 교양영어·영어회화 담당 교강사** (창업팀 연고 = 가장 빠른 첫 1명)
2. **OPIc·토익스피킹 지도 교강사** (말하기 *수행*에 이미 관심 = 고의도)
3. 교양 발표/스피치, 관광·간호 등 **실무 말하기**가 필요한 학과 교양영어
4. 학원·고교 영어회화 강사 (대학 안 되면 대안 채널)

**액션:** 어학교육원 홈페이지/강의편람에서 위 조건 교강사 **이름·이메일 10명**을 표로 뽑는다 → A/B 템플릿으로 이번 주 안에 컨택.

## 6. 킬 기준 (정직하게 멈추는 선)
- 교수 **10명** 만나/연락해서 **0명**이 "과제로 넣겠다"면 → micro-B2B2C 가설 약함.
- 그때는 더 만들지 말고 **좁은 고의도 B2C**(OPIc·면접영어·교환학생 인터뷰)로 포지션 전환 검토.
- 반대로 **1명**이라도 진짜 "예"면 → 그 1개 분반을 *완벽하게* 굴리는 데 올인(다른 기능 다 미룸).
